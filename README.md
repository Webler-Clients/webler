## ğŸš€ How It Works

### 1. Trigger Generation

**Via GitHub Actions UI:**
```yaml
workflow_dispatch:
  inputs:
    business_name: "Acme Corp"
    industry: "Technology"
    services: '["Web Development", "Consulting"]'
    colors: '{"primary":"#3b82f6","secondary":"#8b5cf6"}'
    # ... other inputs
```

**Via API (Convex or other services):**
```bash
curl -X POST \
  -H "Authorization: token $GITHUB_TOKEN" \
  -H "Content-Type: application/json" \
  https://api.github.com/repos/WebAsAService/base-minimal/dispatches \
  -d '{
    "event_type": "generate-client-site",
    "client_payload": {
      "businessName": "Acme Corp",
      "industry": "Technology",
      "services": ["Web Development", "Consulting"],
      "colors": {"primary": "#3b82f6", "secondary": "#8b5cf6"},
      "generationType": "full"
    }
  }'
```

### 2. AI Generation Process

The GitHub Actions workflow:

1. **Initializes workspace** - Sets up Node.js, Python, dependencies
2. **Sends webhooks** - Real-time status updates to caller
3. **Calls Claude API** - Generates complete Astro website from business info
4. **Creates files** - Writes generated `index.astro` to `src/pages/clients/[name]/`
5. **Opens PR** - Creates pull request with generated content
6. **Deploys preview** - Netlify automatically creates preview URL

**Key Innovation**: The AI generates the **entire website** in a single Claude API call - no templates, no scripts, no constraints.

## ğŸ¯ Integration with WaaS Ecosystem

### Convex Integration

The base-minimal repository integrates with Convex SaaS backend:

1. User submits form in `client-portal`
2. Form data sent to `convex-saas` backend
3. Convex triggers `base-minimal` via GitHub repository dispatch
4. Workflow sends status webhooks back to Convex
5. Convex updates database and notifies user

### Routing Logic

```typescript
// In Convex or client-portal
if (generationType === 'full') {
  // Use base-minimal for unconstrained AI generation
  await triggerGitHubWorkflow('base-minimal', payload);
} else {
  // Use base-template for template-based generation
  await triggerGitHubWorkflow('base-template', payload);
}
```
